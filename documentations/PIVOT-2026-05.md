# Architectural Pivot — May 2026 (v0.1 → v0.2)

**Status:** Effective immediately. The original templated 4-layout pipeline is deleted. All planning artifacts in this folder predate the pivot — read this document first to understand which sections are now superseded.

## What changed and why

### v0.1 (superseded) — Templated pipeline

```
prompt
  → Claude picks 4 structural design choices (layout_grid, section_structure,
    hierarchy_pattern, content_density)
  → Claude drafts JSON outline (title + bullets + optional icon per slide)
  → Renderer (python-pptx native, OR pptxgenjs templated, OR WeasyPrint Jinja2)
    composes 4 fixed layouts (single-column / two-up / asymmetric / bento)
  → output file
```

Three fundamental ceiling problems made this architecture incapable of producing
visually-distinctive decks (the project's stated wedge):

1. **One layout per deck.** `content_drafter.py` pinned `choices.layout_grid` once
   and stamped it on every slide. Bento decks were 10 bento slides; single-column
   decks were 10 single-column slides. From a thumbnail, the deck looked templated.
2. **`hierarchy_pattern` was dead code.** No renderer consumed it. number-led /
   image-led / type-led rendered identically.
3. **`style_name` only changed paint.** The 67 ui-ux-pro-max styles collapsed
   to "5 colors + 2 font names." A "Brutalism" style and a "Swiss minimalism"
   style produced identical geometry. The visual variety wedge collapsed to
   colorway-and-fontswap.

### v0.2 (current) — Freeform + visual QA loop

```
prompt
  → LLM (Anthropic API or Claude Code CLI subprocess) writes the deck SOURCE CODE
    directly: complete pptxgenjs JS for .pptx, OR complete HTML+CSS for .pdf
  → execute (Node for .pptx, WeasyPrint subprocess for .pdf, hard timeout)
  → libreoffice / pdftoppm → per-slide JPGs
  → LLM vision inspects each JPG, reports rendering bugs as JSON
  → if bugs: LLM revises the source code, re-execute
  → bounded to 3 iterations (configurable)
```

The LLM has full creative control. Per-slide composition. No fixed layouts.
No JSON intermediate. Theme-appropriate palettes (Claude picks based on subject).
Self-healing via the visual QA loop (bugs the LLM produces get caught and fixed
before final output).

- Architecture Decision 5 (Streamlit native + CSS) → **valid.**

## Roadmap (post-pivot)

1. **Re-incorporate ui-ux-pro-max** as palette/typography presets injected into
   the system prompts. The vendor folder is retained for this purpose. The
   Python loader (`src/ia_pptx/design/library.py`) may be repurposed or
   discarded; TBD.
2. **Image strategy.** Add Unsplash/Pexels API integration so Claude can emit
   `query` fields and the pipeline fetches background imagery for full-bleed
   composition.
3. **Multi-pass polish.** Optional final-pass "polish" prompt that revisits
   the generated source with vision and tightens typography/spacing/contrast,
   mirroring Gamma's "sparkle" agent.

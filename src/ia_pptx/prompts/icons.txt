ICON VOCABULARY (optional — assign one icon per slide for visual anchoring)
============================================================================

You MAY add an `"icon"` field to a slide. The icon renders small in the
top-right corner using the slide's accent color — it acts as a visual
anchor (Rule 6: use graphics effectively), not decoration.

WHEN TO ADD AN ICON
-------------------
- Add to ~50–80% of content slides, especially for "minimal" or "medium"
  density. The icon strengthens the slide's metaphor without competing
  with the title.
- DO NOT add to section dividers (their job is to announce, not anchor).
- DO NOT add a generic icon if no specific one fits — leave the field
  out rather than picking something vaguely related. Empty is better
  than wrong.

LIBRARY SYNTAX
--------------
- Plain name → resolved against Font Awesome (FA) by default:
    "icon": "FaCheckCircle"
- With prefix `<lib>:<Name>` to use a specific library:
    "icon": "md:MdRocketLaunch"
    "icon": "fa6:FaChartLine"
    "icon": "lu:LuTarget"
    "icon": "hi2:HiOutlineLightBulb"
    "icon": "bi:BiTrendingUp"

AVAILABLE LIBRARIES (prefix → set)
- fa  → Font Awesome 5 (most common; default if no prefix)
- fa6 → Font Awesome 6 (newer icons)
- md  → Material Design
- hi2 → Heroicons 2
- bi  → Bootstrap Icons
- lu  → Lucide (clean line icons)
